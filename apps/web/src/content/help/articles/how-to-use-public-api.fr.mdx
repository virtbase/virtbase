L'API publique Virtbase offre un moyen flexible de gérer vos services réservés de manière programmatique. 
Pour cela, vous devez créer une clé API afin d'envoyer des requêtes authentifiées.

## Comment créer une clé API ?

Suivez ces étapes pour créer une clé API.

<Hint>
Les clés API n'ont pas de date d'expiration, mais ne peuvent être affichées qu'une seule fois après leur création.
</Hint>


1. Accédez aux paramètres **Compte** dans le [portail client](https://app.virtbase.com/account/settings).
2. Cliquez sur **API** dans le menu de navigation.
3. Vous voyez alors apparaître une liste de vos clés API déjà actives. Cliquez ici sur **Créer une clé API**.
4. Attribuez un nom à votre clé et confirmez la création.
5. Votre clé API s'affiche alors une seule fois dans la partie supérieure, enregistrez-la soigneusement.

Votre clé API peut être utilisée directement pour les requêtes API. Si vous n'avez plus besoin de votre clé, vous pouvez la supprimer à tout moment.

Si vous avez d'autres questions sur l'API publique ou si vous avez besoin d'aide, [contactez notre équipe d'assistance](/fr/contact).

## Comment utiliser l'API publique ?

Vous trouverez la documentation complète de tous les points de terminaison API à l'adresse suivante : https://virtbase.com/api/docs.

La documentation est disponible uniquement en anglais.

Toutes les requêtes aux points finaux doivent être effectuées avec l'en-tête « X-Virtbase-API-Key ».

## Y a-t-il une limite de débit ?

Oui, vous pouvez envoyer un maximum de 120 requêtes par minute (= 1 requêtes par seconde) à l'API avec une clé API.

Une limite de débit plus élevée n'est possible qu'après autorisation. Pour cela, veuillez contacter notre service d'assistance.
